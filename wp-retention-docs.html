Documentation

Everything you need to install, configure, and get the most out of WP Retention.

Requirements

Before installing WP Retention, make sure your environment meets the following:

  • WordPress 6.0 or higher
  • PHP 7.4 or higher
  • At least one of: MemberPress (any version) or WooCommerce Subscriptions 3.0+
  • WP-Cron enabled, or a real server cron job configured
  • MySQL 5.7+ or MariaDB 10.3+

Note: WP Retention does not work as a standalone plugin. It requires either MemberPress or WooCommerce Subscriptions to be active and configured on your site.

Installation

  1. Download the plugin Download the wp-retention.zip file from your Freemius account after purchase.
  2. Upload via WordPress admin Go to Plugins → Add New → Upload Plugin, select the zip file and click Install Now.
  3. Activate Click Activate Plugin. WP Retention will automatically create its database table and a cancel flow page in your WordPress install.
  4. Configure Go to WP Retention → Settings to adjust scoring weights, trigger thresholds, and email templates.

On activation, the plugin creates a page titled "Cancel Subscription" with the shortcode [wpr_cancel_flow]. Do not delete this page — it powers the cancel flow interception.

License activation

After purchase, Freemius sends you a license key by email. To activate:

  1. Go to WP Retention in your dashboard You'll see a license activation prompt on first use.
  2. Enter your license key Paste the key from your purchase confirmation email.
  3. Click Activate The plugin connects to Freemius to validate and unlock all features.

Your license allows installation on 1 site (Solo) or up to 5 sites (Agency). To move a license to a different site, deactivate it first from the old site via WP Retention → Account.

Scoring weights

WP Retention calculates a risk score from 0 to 100 for every active subscriber. A score of 0 means fully engaged; 100 means imminent churn.

The score is composed of four weighted signals. You can adjust the weight of each signal in Settings → Scoring Weights. Weights must total 100 — they are normalised automatically if they don't.

Signal Default weight What it measures
Login inactivity 35% Days since the subscriber last logged in. No login ever recorded scores 80/100 on this component.
Content inactivity 25% Number of protected content views in the last 90 days. Zero views scores 95/100 on this component.
Payment failures 30% Number of failed payment events in the last 90 days. Two or more failures scores 100/100 on this component.
Subscription age 10% New subscribers (under 30 days) carry a higher baseline risk. Subscribers over 90 days score 0 on this component.

Recommended weight profiles

Content-heavy membership site (courses, LMS): raise Content inactivity to 40%, lower Login inactivity to 25%.

SaaS-style tool: raise Payment failures to 40%, lower Subscription age to 5%.

Newsletter / community: raise Login inactivity to 50%, lower Content inactivity to 15%.

Triggers

When a subscriber's score reaches or exceeds the Alert Threshold (default: 60), WP Retention automatically fires a trigger action.

Three trigger modes are available in Settings → Trigger Settings:

  • Email only — sends a personalised retention email via wp_mail()
  • Coupon only — generates a unique WooCommerce coupon tied to the subscriber's email (requires WooCommerce)
  • Email + Coupon — both actions simultaneously

A cooldown period (default: 30 days) prevents the same subscriber from being triggered more than once per period. Adjust this in Settings to avoid over-solicitation.

Cancel flow

WP Retention intercepts the default cancellation button in MemberPress and WooCommerce Subscriptions and redirects subscribers to a custom cancel page.

The flow has three steps:

  1. Reason selection — the subscriber picks a cancellation reason from up to 5 options (configurable)
  2. Conditional offer — an offer is presented based on the selected reason (discount, pause, or downgrade)
  3. Confirmation — the subscriber either accepts the offer or confirms cancellation

Default reason-to-offer mapping

ReasonOffer typeAction
It's too expensiveDiscountGenerates a % coupon and emails it
I need a breakPauseSets subscription to on-hold for 30 days, schedules reactivation
Not getting enough valueDowngradeFlags user for admin follow-up, no automatic action
Finished what I neededNoneProceeds to cancellation
Another reasonNoneProceeds to cancellation

You can customise the reasons and their labels using the wpr_cancellation_reasons filter hook.

Email template

The retention email is sent via wp_mail() — no external email service required. Configure the subject and body in Settings → Retention Email Template.

Available template variables:

  • {first_name} — subscriber's first name
  • {site_name} — your WordPress site name
  • {login_url} — your WordPress login URL
  • {offer_text} — automatically replaced with the coupon code if a coupon was generated, or a generic message if not

Developer hooks

WP Retention exposes several filters and actions for developers who need to customise behaviour.

Filters

// Customise cancellation reasons and their offer types
add_filter( 'wpr_cancellation_reasons', function( $reasons ) {
    $reasons['technical'] = [
        'label' => 'Technical issues',
        'offer' => 'discount',
    ];
    return $reasons;
} );

// Customise which post types are tracked for content views
add_filter( 'wpr_tracked_post_types', function( $types ) {
    $types[] = 'podcast_episode';
    return $types;
} );

Actions

// Fire after a risk score is calculated for a user
add_action( 'wpr_score_calculated', function( $user_id, $score ) {
    // Your custom logic here
}, 10, 2 );

// Fire after a trigger action is sent
add_action( 'wpr_trigger_fired', function( $user_id, $score, $action ) {
    // Log to external system, etc.
}, 10, 3 );

Cron & scoring

WP Retention calculates risk scores once per day via WP-Cron, scheduled at 2:00 AM server time. Scoring runs in batches of 50 subscribers to avoid PHP timeouts.

On shared hosting (OVH, Ionos, etc.), WP-Cron only fires when someone visits your site. For reliable daily scoring, configure a real server cron job:

# Add to crontab (crontab -e)
0 2 * * * php /path/to/your/wordpress/wp-cron.php --allow-root > /dev/null 2>&1

And disable WordPress's built-in pseudo-cron in wp-config.php:

define( 'DISABLE_WP_CRON', true );

To trigger scoring manually, go to WP Retention → At-Risk and click "Run scoring now". Individual subscribers can be re-scored using the Re-score button in the dashboard table.

Database

WP Retention creates one custom table: {prefix}wpr_events. All other data is stored in wp_usermeta under the _wpr_ prefix.

FAQ

Does the plugin work with both MemberPress and WooCommerce Subscriptions simultaneously?

Yes. If both plugins are active, WP Retention hooks into both and merges subscriber IDs for scoring. A subscriber tracked via both plugins is de-duplicated automatically.

What happens if WP-Cron doesn't run for several days?

Scores simply aren't recalculated. Existing scores remain until the next cron execution. No data is lost. Use the "Run scoring now" button to manually trigger a recalculation.

Can I exclude certain subscribers from scoring?

Not via the admin UI in v1.0. Developers can use the wpr_cancellation_reasons filter or hook into wpr_score_calculated to implement custom exclusion logic.

Does the pause feature work with MemberPress?

Pausing via the cancel flow currently only works with WooCommerce Subscriptions (which supports on-hold status natively). For MemberPress, the downgrade offer is used instead. Full MemberPress pause support is planned for v1.1.

How do I uninstall cleanly?

Deactivate the plugin, then delete it via the WordPress plugin screen. The uninstall routine automatically drops the wpr_events table, removes all _wpr_* usermeta, deletes the cancel flow page, and removes all plugin options.
