Troubleshooting

Common issues.

The plugin activated but I don't see the WP Retention menu
Make sure you have either MemberPress or WooCommerce Subscriptions active on the same site. WP Retention requires at least one of them to boot correctly. Check your PHP error log for any fatal errors on activation.
Scores are all showing 0 or haven't updated
Scoring runs via WP-Cron once per day. If your site has low traffic, WP-Cron may not be firing. Go to WP Retention → At-Risk and click "Run scoring now" to trigger a manual recalculation. For long-term reliability, configure a real server cron job — see the documentation.
The cancel flow page isn't showing — subscribers can still cancel directly
Check that the page with the [wpr_cancel_flow] shortcode exists and is published (go to Pages and search "Cancel Subscription"). Also verify that the option wpr_cancel_page_id is set in your WordPress options table. If the page was accidentally deleted, deactivate and reactivate the plugin — it will recreate the page on activation.
Retention emails aren't being sent
WP Retention uses wp_mail() which relies on your server's email configuration. If emails aren't arriving, install an SMTP plugin (WP Mail SMTP, Brevo, Postmark) and configure it — most shared hosting servers have poor email deliverability without SMTP. Also check that the trigger threshold in Settings is set appropriately and that the cooldown period hasn't blocked re-triggering.
WooCommerce coupon isn't being generated
Coupon generation requires WooCommerce to be active (not just WooCommerce Subscriptions). Confirm that the WooCommerce plugin itself is active and that your WooCommerce version is 5.0 or higher. The coupon discount percentage is configurable in Settings → Trigger Settings → Coupon Discount.
Subscribers are being triggered too frequently
Increase the Cooldown Period in Settings (default: 30 days). This prevents the same subscriber from triggering more than once per period. Also consider raising the Alert Threshold to only act on subscribers with very high risk scores.
Fatal error after activation: class not found
This typically means the plugin's PHP files aren't loading in the right order. Make sure you're running PHP 7.4 or higher. If the error persists, check that the zip was fully extracted and no files are missing — try reinstalling from a fresh download.
The pause feature doesn't work with MemberPress
Subscription pause (on-hold) is currently only supported for WooCommerce Subscriptions, which has native on-hold status. For MemberPress subscribers, the cancel flow presents a downgrade offer instead. Full MemberPress pause support is planned for v1.1.
Before you submit a ticket

Help us help you faster.

Including this information in your support request cuts resolution time in half:

  • WordPress version and PHP version
  • MemberPress version and/or WooCommerce Subscriptions version
  • The exact error message (copy from PHP error log or browser console)
  • Steps to reproduce the issue
  • Whether the issue occurs with all other plugins deactivated
